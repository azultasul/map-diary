# Map Visual Spec v0.1

## 1. 목적

Map Diary의 지도 화면은 단순한 위치 표시가 아니라 사용자의 여행 기록을 감성적으로 보여주는 핵심 UI이다.

지도 화면의 목표는 다음과 같다.

* 여행 경로를 직관적으로 보여준다.
* 사용자의 기록이 쌓이는 느낌을 제공한다.
* 3D 지구본과 2D 지도가 동일한 데이터를 기반으로 자연스럽게 전환된다.
* 첫 화면에서 세련되고 기억에 남는 인상을 준다.
* 사용자가 원하는 지역을 자유롭게 탐색할 수 있다.

---

## 2. 전체 디자인 방향

키워드

* 세련된
* 어두운
* 미니멀한
* 공간감 있는
* 항공 경로 같은
* 개인 기록 아카이브 같은

기본 테마

* 다크 모드 우선
* 지구/지도는 과하게 현실적인 위성 지도보다 추상화된 스타일
* 핀과 경로는 여행 기록이 강조되도록 밝고 선명하게 표현

### 지도 인터랙션

3D Globe와 2D Map 모두 사용자가 자유롭게 탐색할 수 있어야 한다.

지원 기능

* 마우스 드래그 또는 터치 드래그를 통한 이동
* 마우스 휠 또는 핀치 제스처를 통한 줌인/줌아웃
* 특정 도시 또는 핀 선택 시 해당 위치로 카메라 이동
* 사용자가 마지막으로 보고 있던 위치와 줌 레벨 유지

## 3. 3D Globe Mode

### 지구본 스타일

* 어두운 네이비 또는 블랙 계열 구체
* 대륙은 은은한 라인 또는 면으로 표시
* 바다는 깊은 어두운 색감
* 국가 경계는 너무 강조하지 않고 은은한 라인으로 표시
* 대륙 경계 또는 육지 윤곽만 표시해도 충분

### 배경

* 어두운 그라데이션 배경
* 은은한 별 또는 입자 효과
* 지나치게 화려한 우주 배경은 지양

### 카메라 및 탐색

사용자는 지구본을 자유롭게 조작할 수 있다.

지원 기능

* 지구본 회전
* 줌인
* 줌아웃
* 관성(Inertia) 기반 부드러운 회전
* 최소/최대 줌 레벨 제한

도시 선택 시

* 해당 도시가 화면 중앙에 오도록 카메라 이동
* 현재 선택된 도시 강조

### 도시 핀

기본 상태

* 작은 원형 점
* 그룹 색상 사용
* 약한 glow 효과

호버 상태

* 점 크기 확대
* 도시명 툴팁 표시
* 해당 도시와 연결된 경로 강조

선택 상태

* 핀 pulse 애니메이션
* 해당 도시의 일기 목록 모달 표시

### 이동 경로

* 도시 간 이동은 곡선 arc로 표시
* 날짜 순서 기준으로 연결
* 그룹 필터 적용 시 해당 그룹의 경로만 표시
* 경로 색상은 그룹 색상 사용
* 전체 보기 상태에서는 여러 그룹 색상이 동시에 보일 수 있음

### 경로 애니메이션

MVP 기본

* 정적인 곡선 표시

PoC 이후 추가

* 경로를 따라 작은 빛 점이 이동
* 경로가 순차적으로 그려지는 애니메이션
* 도시 클릭 시 해당 도시로 카메라 이동

## 4. 2D Map Mode

### 지도 스타일

* Three.js 기반 평면 세계 지도
* 완전한 2D보다 약간의 입체감과 깊이감 제공
* 다크 톤 배경
* 대륙은 낮은 높이의 extrude 또는 얇은 면으로 표현
* 지도 가장자리는 부드럽게 fade 처리 가능

### 카메라 및 탐색

사용자는 평면 지도를 자유롭게 탐색할 수 있다.

지원 기능

* 드래그를 통한 지도 이동
* 줌인
* 줌아웃
* 최소/최대 줌 레벨 제한
* 도시 선택 시 해당 위치로 부드럽게 이동

줌 레벨에 따른 동작

* 낮은 줌: 전체 세계 지도 중심
* 중간 줌: 국가 단위 탐색
* 높은 줌: 도시 단위 탐색

### 도시 핀

* 3D 지구본과 동일한 데이터 사용
* 그룹 색상 기반 핀
* 핀 클릭 시 도시 일기 목록 모달 표시

### 이동 경로

* 도시 간 경로는 곡선 또는 부드러운 라인으로 표시
* 화살표 또는 moving dot으로 방향성 표현
* visitedDate 기준으로 연결

## 8. 데이터 표시 규칙

### 도시 핀 생성 기준

diaries 데이터를 city, country 기준으로 그룹화한다.

같은 도시에 여러 개의 일기가 있으면 하나의 핀만 표시한다.

핀에는 diaryCount를 기본적으로 표시한다.

### 경로 생성 기준

기본 경로는 diaries를 visitedDate 오름차순으로 정렬하여 생성한다.

같은 도시에서 여러 개의 일기가 연속으로 있을 경우, 중복 경로는 생성하지 않는다.

예시

Tokyo → Tokyo → Osaka

실제 경로

Tokyo → Osaka

### 그룹 필터 적용 시

선택된 groupId의 diaries만 사용한다.

그 후 도시 핀과 경로를 다시 계산한다.

## 9. PoC 성공 기준

### 3D Globe PoC

* 지구본이 렌더링된다.
* 사용자가 지구본을 회전할 수 있다.
* 사용자가 줌인/줌아웃할 수 있다.
* mock-data.json의 도시가 핀으로 표시된다.
* 도시 핀 클릭 시 도시명이 출력된다.
* 도시 클릭 시 해당 도시의 일기 목록을 확인할 수 있다.
* visitedDate 기준으로 도시 간 경로가 표시된다.
* 그룹 필터 적용 시 해당 그룹 데이터만 표시된다.

### 2D Map PoC

* 평면 세계 지도가 렌더링된다.
* 사용자가 지도를 이동할 수 있다.
* 사용자가 줌인/줌아웃할 수 있다.
* 동일한 mock-data.json을 사용한다.
* 도시 핀과 이동 경로가 표시된다.
* 그룹 필터가 동일하게 동작한다.

## 10. 구현 우선순위

1. mock-data.json 연결
2. diaries → cityMarkers 변환 함수 작성
3. diaries → routes 변환 함수 작성
4. 3D Globe 렌더링
5. 3D Globe 카메라 컨트롤(회전, 줌인, 줌아웃)
6. 도시 핀 표시
7. 경로 arc 표시
8. 도시 클릭 이벤트
9. 그룹 필터
10. 2D Map 렌더링
11. 2D Map 카메라 컨트롤(이동, 줌인, 줌아웃)
12. 2D Map에서도 동일 데이터 표시
